# 01. 백엔드 최초 셋업 및 실행

> ※ 백엔드 코드는 **빌드 단계에서 생성**됨. 이 매뉴얼은 그 후 동작하는 절차이고, 빌드는 이 절차대로 맞춰 만든다.

**전제**: `00_사전_준비물_설치.md` 완료 (uv 설치 + Docker Desktop 실행 중).

---

## 처음 한 번 (순서대로)

### 1) 백엔드 폴더로 이동
```powershell
cd D:\work\project\workspace\Prj\stockbot\backend
```

### 2) `.env` 만들기
템플릿 복사 후 값 채우기 (값 설명은 **`02_환경변수_env_설정.md`** 참고):
```powershell
Copy-Item .env.example .env
# 그 다음 .env 열어서 값 입력 (PAIRING_KEY, TOSS_KEY_ENC_KEY 등)
```

### 3) Postgres 띄우기 (도커)
```powershell
docker compose up -d
```
→ Postgres 컨테이너가 백그라운드로 뜸. (확인: `docker ps`)

### 4) 의존성 설치 + 가상환경 준비
```powershell
uv sync
```
→ `.venv` 자동 생성 + lock대로 라이브러리 설치. (수동 venv 불필요)

### 5) DB 마이그레이션 적용 (테이블 생성)
```powershell
uv run alembic upgrade head
```
→ `credentials` 테이블 등 스키마 생성.

### 6) 서버 실행
```powershell
uv run uvicorn stockbot.main:app --reload --port 8000
```

### 7) 동작 확인
- 브라우저: http://localhost:8000/health  → 정상 응답
- API 자동 문서: http://localhost:8000/docs

---

## 종료
```powershell
# 서버: 터미널에서 Ctrl + C
docker compose down      # Postgres 내리기 (데이터는 볼륨에 유지)
```

## 이후 다시 실행할 때 (이미 셋업 끝난 뒤)
```powershell
docker compose up -d                                   # (안 떠있으면)
uv run uvicorn stockbot.main:app --reload --port 8000  # 서버
```
- 라이브러리/마이그레이션이 바뀌었을 때만 `uv sync` / `uv run alembic upgrade head` 추가.

---

## 내가(사용자) 하는 것 vs 자동
- **내가**: 위 명령 실행 + `.env` 값 입력 + Docker Desktop 켜기.
- **자동(uv가)**: 가상환경 생성·라이브러리 설치·파이썬 버전 맞추기.
