# 뉴스·공시 소스 Feasibility (조사 결과)

- **조사일**: 2026-06-24 (웹 조사 기반)
- **대상**: 국내+미국, 1인 개인 봇, 무료 우선
- **한 줄 결론**: 핵심 뉴스·공시 수집은 **무료 소스만으로 가능**. 단 *무료로 진짜 per-ticker 한국뉴스는 없어서* 회사명 매칭(노이즈)을 감수하고 별칭 사전으로 보완해야 함. 공시(DART/EDGAR)는 무료·공식·실시간 1차 사실로 **최고 신호**.

## 3. 핵심 현실 제약 (설계에 반영)

1. **무료 per-ticker 한국뉴스 없음** — 네이버/RSS는 회사명 매칭이라 노이즈('삼성'→여러 계열사). → **별칭 사전 + 스톱리스트** 필수. 종목코드(005930) 검색은 안 됨.
2. **본문 저장 금지** — 메타 API는 헤드라인+짧은 요약만 줌. 본문 전체 저장/재배포는 저작권 위험 → 우리는 *요약(우리 말로)·링크*만 저장. (상세: `design/news-pipeline.md`)
3. **공시는 "발생 사실+제목"만** — 의미 해석은 우리 LLM/규칙 몫. 조인키(corp_code/CIK) 사전 구축 필요.
4. **신선도 병목은 우리 폴링 주기** — SEC 1초미만/DART 분단위/네이버 준실시간이라, 늦는 건 API가 아니라 우리가 얼마나 자주 도느냐.
5. **rate limit·UA·RSS 불안정** — Alpha Vantage 25/일, GDELT 429 가능, SEC User-Agent 필수, KR RSS URL 조용히 바뀜 → health-check+폴백.

## 4. 유료 escalation (미래 옵션, 지금 불채택)

- **DeepSearch 뉴스 API** — 한국 ticker 뉴스 + AI요약 + 등락사유까지. 무료 per-ticker 한국뉴스의 부재를 정공법으로 해결. *통제권 욕구상 지금은 직접 구축이 맞아 불채택*, 나중에 KR 뉴스 품질이 부족하면 보강 후보.
- Marketaux / Polygon(Massive) Starter(~$29/mo) — finance-native 태깅+감정 at volume.
