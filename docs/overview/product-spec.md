# Stock Bot — 개인 AI 주식 비서 설계 문서

- **문서 버전**: v1 (브레인스토밍 결과)
- **작성일**: 2026-06-24
- **상태**: 설계(기획) 단계 — 구현 전. 사용자 검토 후 구현 계획서로 진행.
- **사용자**: 단독 사용 (1인). 국내+미국 주식 투자자. 직장인.

---

## 1. 목적 (Purpose)

> 바쁜 직장인이라 직접 못 보는 **국내+미국 시장 정보**를, 봇이 24시간 **수집·분석**해서
> → 정해진 시각 **리포트(앱)** 로 정리하고
> → 장중 중요한 **급변/뉴스/이슈는 즉시 디스코드 알림**으로 주고
> → 물어보면 답하고 내 계좌를 분석해주는 **AI 비서**로서,
> **최종 판단은 내가 더 잘 하도록 돕는다.**

즉 단순 리포터가 아니라 **개인 AI 주식 비서** — 능동(채팅·포트폴리오 분석) + 수동(수집·분석·보고·알림) 둘 다.

## 2. 핵심 원칙 (Principles)

이 원칙들이 모든 설계 결정의 기준이다.

1. **추천·자동매매 아님 (지금). 판단·실행은 나.** — 봇은 정보·분석·근거를 제공하고, 매매 결정은 사용자가 한다. (단, 자동매매는 검증 후 미래 확장으로 명시 — §10)
2. **정직한 raw 데이터 > 클레버한 추측.** — "왜 움직였나"는 항상 *가설*이지 사실이 아니다. 모르면 "모른다"고 말한다. 신뢰도와 근거 원문을 항상 함께 보여준다.
3. **통제권은 내가 쥔다.** — 외부 올인원 분석 API에 의존하지 않는다. RAW 데이터만 외부(무료)에서 받고, 분석 에이전트는 직접 만든다. LLM만 외부(Claude). 무거운 프레임워크(LangChain 등)에 종속되지 않는다.
4. **1인 기술 사용자 기준.** — UI로 만들 필요 없는 것은 코드/파일 경로로 간다(예: 정정). 앱은 *보는* 용도 중심.
5. **단계적 가치, 견고한 아키텍처.** — 빅뱅으로 다 만들지 않는다. 심장(데이터·분석 파이프라인)을 제대로 깔고 가치를 단계적으로 올린다.

## 3. 범위 (Scope)

### In scope
- 토스 계좌 연동(읽기 전용): 보유종목·시세
- 관심종목 직접 관리 (토스에 없는 기능)
- 무료 외부 소스에서 뉴스·공시 수집 (국내+미국)
- 직접 구축한 분석 파이프라인 (LLM = Claude)
- 일일(2회+커스텀)·주간 리포트 → **앱**
- 장중 긴급 알림(급변·공시·목표가/손절가·실적임박) → **디스코드**
- 채팅 비서 (앱), 포트폴리오 AI 분석
- 목표가/손절가 도달 알림 (알림만, 읽기 전용)
- 테마 바스켓 추적 + 연관·공급망 레이더
- 실적·배당 이벤트 사전 알림
- 정정/길들이기(정정 지식파일 → 분석 프롬프트 주입)

### Out of scope (지금은 안 함 / 보류)
- 자동매매 실행 (미래 확장 — §10에 설계만 준비)
- 종목 추천·미래 예측기(RAG-as-predictor)
- 52주 신고가/저가 알림, 환율 임계 알림, 공시 키워드 트립와이어 (보류 — 원하면 추가)
- 신규 테마 부상 감지(투기적 — 후순위)
- 전용 벡터DB(Pinecone 등) — 필요해지면 pgvector만 얹음
- 이메일 채널 (앱+디스코드로 충분)
- 다중 사용자

## 5. 외부 의존성 & Feasibility (조사로 검증됨, 2026-06-24)

### 5.1 토스 Open API (읽기 전용)
- **인증**: OAuth2 Client Credentials (client_id/secret 붙여넣기, 로그인 리다이렉트 없음). **refresh token 없음 → 24h마다 재발급**. 토큰 만료 전 선제 재발급.
- **시세 폴링**: `/prices`가 **한 번에 200종목** → 관심+보유 ~30종목이 **1콜/폴링**. 5~15초 주기도 여유. 토큰 1개 소모.
- **미국 주식 완전 지원**: KR=6자리숫자(005930), US=티커(AAPL). 보유종목도 KR/US(원화·달러 분리) 제공.
- **rate limit**: 그룹별 token-bucket, 429 시 `X-RateLimit-*`/`Retry-After` 헤더. **정확한 숫자는 스펙에 없음 → 429 적응형 백오프** 구현. (헤더 예시값 10은 비보장)
- **캘린더·환율**: 국내/미국 장 운영 캘린더, KRW/USD 환율(1분 갱신) 제공.
- **❗ 토스가 안 주는 것**: 뉴스, 공시, **전체 종목 목록/검색**, 잔고 히스토리, websocket(폴링만), 분/일봉 외 캔들, 펀더멘털/배당.
- **⚠️ 보안 사실**: API 키에 scope가 없어(`{}`) **같은 키로 조회·주문 다 가능.** "읽기 전용"은 키가 아니라 *우리 코드*가 지킨다 → §10 안전 플래그 필수.

### 5.2 뉴스 소스 (전부 무료)
| 시장 | 소스 | 비고 |
|---|---|---|
| 🇰🇷 | **네이버 검색 API(뉴스)** | 무료 2.5만콜/일, 준실시간, ToS 합법(스크래핑X), 키 붙여넣기. 회사명 검색=의사 per-ticker |
| 🇰🇷 | **증권 RSS**(한경/매경/파이낸셜뉴스 stock.xml) | 무료 firehose → 회사명 필터. URL 불안정 → health-check+폴백 |
| 🇺🇸 | **Finnhub** company-news | 무료 60콜/분, 종목 태깅됨(매칭 해결). 감정은 유료 → 자체 처리 |
| 🌐 | **GDELT** | 무료·키없음·감정점수 내장. 거시/분위기. ~15분, 3개월 윈도우 |
| 🌐 | **Alpha Vantage** NEWS_SENTIMENT | 무료 25콜/일(짠 편) → 하루 1~2회 거시+감정 다이제스트 |

- **⚠️ 정직한 한계**: *무료로 진짜 per-ticker 한국뉴스는 없다.* 회사명 매칭은 노이즈('삼성'→여러 계열사) → **별칭 사전 + 스톱리스트** 필요. 본문 저장/재배포 금지(헤드라인·요약·링크만).
- **유료 escalation(미래 옵션)**: DeepSearch 뉴스 API(한국 ticker 뉴스+AI요약). 지금은 불채택.

### 5.3 공시 (무료, 1차 사실 — 최고 신호)
- 🇰🇷 **DART OpenAPI**: 무료, 금감원 공식, **분 단위 실시간**. 공시검색 폴링→`rcept_no` diff→신규 알림. 주요사항보고(유상증자·CB/BW·공급계약·최대주주변경·횡령), 지분공시(5%룰). 조인키=8자리 corp_code(corpCode.xml 매핑).
- 🇺🇸 **SEC EDGAR submissions API**: 무료·키없음·**1초 미만 실시간**. ticker→CIK(company_tickers.json) 후 폴링→`accessionNumber` diff→8-K/10-Q/Form4 알림. (full-text search는 키워드 트립와이어용, 보류)
- **⚠️**: 공시는 "일어났다"는 사실+제목만 줌. *의미 해석*은 우리 LLM/규칙 레이어 몫. SEC는 User-Agent 헤더 필수(없으면 차단).

### 5.4 종목 마스터
- 🇰🇷 **KRX** 또는 **DART corpCode** → 전체 KOSPI/KOSDAQ 코드+이름. 🇺🇸 **SEC company_tickers.json** → 전체 티커+회사명+CIK. 주 1회 갱신.
- 한 데이터셋이 ① 관심종목 검색 ② 뉴스 종목매칭 별칭 ③ 공시 조인키 세 가지를 해결.

### 5.5 AI (Claude)
- **Haiku 4.5**: 대량 1차 분류·매칭 잔차 ($1/$5 per MTok, 배치 50%).
- **Opus 4.8**: 깊은 분석·리포트·채팅·포트폴리오 ($5/$25, 예전보다 저렴). 4.7+ 토크나이저는 토큰 ~35%↑ → 버퍼.
- **예상 비용**: 월 **$30~50** (채팅·포트폴리오 포함). 프롬프트 캐싱으로 보유/스타일 블록 절감.

### 5.6 채널
- **디스코드 웹훅**: URL 1개, OAuth/토큰 갱신 없음, 즉시 푸시, 임베드. 알림 전용으로 충분(채팅은 앱이라 봇 게이트웨이 불필요).
- **앱**: 자체. 리포트·대시보드·설정·채팅·포트폴리오.

### 5.7 ⚠️ 빌드 시 검증 필요 항목
- KR 실적·배당 캘린더 정확한 소스 (현실적으로 DART 잠정실적 공시 기반 추정 + 공시 시 즉시)
- 토스 실제 rate limit 수치 (개발자 콘솔/실측)
- 테마 멤버십·공급망 관계 시드 데이터 (큐레이션 지식파일 + Opus 제안)
- 앱 폼팩터(웹 vs 모바일) — 기술 선정 단계에서 확정

## 6. 기능 상세 (동작 설계)

**1) 종목 마스터 & 관심종목** — KRX+SEC import(주1회) → `종목마스터`. 앱에서 이름 검색→관심종목 추가/삭제.

**2) 보유 동기화 & 손익 히스토리** — `/holdings` 주기 수집 → `보유스냅샷` 매일 적재. 우리 스냅샷이 곧 손익 타임라인(포트폴리오 분석·차트 토대).

**3) 수집→매칭→1차분류** — Ingestor가 무료 소스 RAW→`원본아이템` 정규화. Entity Matcher(소스태그→별칭사전→Haiku)→`아이템-종목링크`. Tier0 무료필터(중복·무관 제거)→Haiku 1차분류(관련/감정/신규/한줄요약/escalate). 대부분 여기서 저장만.

**4) 깊은 분석 엔진 (Opus 4.8 + 안전장치 4종)** — escalate/이상신호 입력 + 보유맥락 + 가격맥락 + **과거 같은종목 N건(SQL)** + **정정 지식파일** 주입. 출력=`분석`(무슨일/왜/내종목 의미 + 신뢰도 + 근거원문).
  - 안전장치 ① 시장/섹터 베타 잔차 먼저(잔차≈0 → "그냥 시장 따라감") ② 뉴스 타임스탬프가 움직임보다 *먼저*인지 검증 ③ "원인 불명" 당당히 허용 ④ 수치·인명은 원문에 있는 것만(없으면 인용 거부).

**5) 급변·공시 감지 & 알림** — Anomaly Detector(가격±N%·거래량 배수·베타잔차) + DART/EDGAR 신규공시 tripwire → 임계 초과 시 4)로 "왜?" 붙여 **디스코드**. mute 중이면 보류, `critical`(유상증자·횡령 등)은 뚫고 나감.

**6) 목표가/손절가 알림 (A: 알림만, 읽기 전용)** — 보유종목별 목표가·손절가(또는 손절 %)를 설정/지식파일에 입력 → 닿으면 즉시 디스코드. **매매는 사용자가 토스앱에서.** (미래 자동매매 확장은 §10)

**7) 테마 바스켓(a) + 연관·공급망 레이더(c)** — 테마(반도체 등) 묶음 종합 등락·뉴스 추적. 미보유라도 내 종목 영향 종목(엔비디아→하이닉스) 뉴스 추적. 멤버십·관계는 **큐레이션 지식파일 + Opus 제안**(무료 데이터 없음). ※ "테마 vs 종목 구분"은 4)의 베타잔차로 이미 제공.

**8) 실적·배당 이벤트 캘린더** — 보유/관심 종목 실적발표일·배당락일 추적 → 며칠 전 사전 알림. US 신뢰도↑, KR은 DART 기반(빌드때 검증).

**9) 일일·주간 리포트 (앱)** — Scheduler 시각(일 2회+커스텀, 주 1회)에 그 사이 쌓인 분석·중요뉴스·포트폴리오 변화를 Opus가 **읽기 쉬운 한국어 리포트**로(헤드라인→쉬운 설명→마무리 깊은 분석) → 앱에 게시. 디스코드엔 핵심 결론만 푸시.

**10) 채팅 비서 (Opus 4.8, 앱)** — Memory(뉴스·공시·분석·스냅샷)+실시간 토스(현재가·보유) retrieval + Claude 네이티브 tool-use로 대화형 답변 + 근거. 정정 입력 경로 중 하나로도 사용 가능하나, 기본 정정은 코드/지식파일(§16).

**11) 포트폴리오 AI 분석 (Opus 4.8)** — 보유스냅샷 기반 시장/섹터/통화 비중, 집중도·리스크, 손익추이 + Opus 코멘트. 앱 화면 + 리포트 한 섹션.

**12) 정정/길들이기** — 종목별 정정 지식파일(`knowledge/<symbol>.md`, git) 직접 편집 → 분석·채팅 프롬프트에 맥락 주입. Entity Matcher가 새 별칭 발견 시 사전 자동 추가. **예측기가 아니라, 쓸수록 나에게 맞춰지는 분석기.**

## 7. 데이터 모델 (Memory)

관계형 DB + 전문검색(FTS). 주요 엔티티:

- `종목마스터(stock_universe)`: symbol, market(KR/US), name, aliases[], corp_code(DART), cik(SEC), sector, theme[]
- `관심종목(watchlist)`: symbol, added_at
- `보유스냅샷(holding_snapshot)`: date, symbol, qty, avg_price, market_value, currency, pnl
- `목표가설정(price_target)`: symbol, target_price, stop_price(or stop_pct), enabled
- `원본아이템(raw_item)`: id, source, type(news/filing), title, url, summary, published_at, fetched_at, source_tickers[]
- `아이템종목링크(item_ticker)`: item_id, symbol, confidence, matched_by(tag/dict/llm), evidence_span
- `분류(triage)`: item_id, relevance, sentiment, novelty, one_line, escalate
- `분석(analysis)`: id, symbols[], trigger(news/filing/anomaly/target/manual), body, confidence, evidence_refs[], model, created_at
- `이상이벤트(anomaly_event)`: symbol, ts, pct_move, volume_ratio, beta_residual, flagged
- `알림(alert)`: id, symbol, type, severity, body, channel, sent_at, muted
- `이벤트(calendar_event)`: symbol, type(earnings/dividend), date, source, confidence
- `테마(theme)` / `테마멤버(theme_member)` / `공급망관계(relation)`: 큐레이션
- `설정(settings)`: sensitivity, mute_until, report_schedule[], TRADING_ENABLED
- `채팅(chat_session/chat_message)`
- 정정은 DB가 아닌 **git 지식파일**(`knowledge/<symbol>.md`)로 관리, 로딩 시 주입.

## 8. AI 파이프라인 & 모델 정책

- **Tier 0 (무LLM 무료필터)**: URL/제목 dedup + 별칭 미포함 제거 → 50~80% 컷.
- **Tier 1 (Haiku 4.5)**: 관련/감정/신규/요약/escalate JSON. 배치(50%↓). 시스템+watchlist 프롬프트 캐싱.
- **Tier 2 (Opus 4.8)**: escalate된 것·이상신호의 깊은 분석 + 리포트 + 채팅 + 포트폴리오. 보유/스타일/정정 블록 캐싱.
- **라우팅**: Tier0 → Haiku(전량 싸게) → escalate(고관련+고신규+보유연관) → Opus. *비싼 모델은 firehose를 안 본다.*
- 안전장치(§6-4)는 분석 프롬프트에 강제. 모든 인과는 가설로 표기 + 근거·신뢰도 첨부.

## 9. 알림·리포트 정책

- **📱 앱**: 모든 리포트(일일 2회+커스텀, 주간) + 대시보드 + 설정 + 채팅 + 포트폴리오.
- **💬 디스코드**: 뉴스 분석 *결론* + 모든 긴급 알림(급변·공시·목표가/손절가·실적임박).
- **민감도**: 기본 "보통", 설정으로 조정. **mute**(N시간/오늘) 지원, 단 `critical`은 mute 무시하고 발송.
- **장 시간대 인지**: KR/US 장 운영 캘린더로 의미 없는 시간대 폴링·알림 억제.

## 10. 자동매매 로드맵 (미래 확장)

**지금**: A(알림만, 읽기 전용). **미래**: 로직 검증 후 B(승인 후 실행) 또는 C(자동 실행)로 확장. 지금부터 확장을 염두에 둔 설계:

- 🔒 **읽기 전용 코어**: 주문 엔드포인트 코드에서 미호출.
- 🚩 **하드 안전 플래그 `TRADING_ENABLED=false`**: 토스 키가 scope 없이 주문도 가능하므로, 코드 레벨에서 차단. 자동매매는 이 플래그 + 명시적 승인장치 통과 시에만.
- 🧩 **알림→(미래)실행 경계 분리**: 지금은 "목표가 닿음→알림"까지. 미래엔 그 경계에 *격리된 실행 모듈*을 끼움(코어 재작성 없이).
- 🔐 **자격증명 분리·승인 게이트**: 자동매매 도입 시 별도 운영·승인 절차 재설계.

## 11. 운영 (Operations)

- **런타임**: 클라우드 VM(Oracle/GCP) 24h. Scheduler가 폴링·리포트·소스 주기 관리(장 시간대 인지).
- **토큰**: 토스 24h 재발급(선제). 네이버/Finnhub/DART 키 보관.
- **회복력**: 429 적응형 백오프(Retry-After), RSS URL health-check+폴백, 소스 다운 시 다른 소스로 graceful degrade, LLM 실패 시 재시도/스킵.
- **시크릿**: 토스 client_id/secret, Claude 키, 디스코드 웹훅 URL, 뉴스 API 키 — VM 안전 보관(환경변수/시크릿).
