# 01. 백엔드 최초 셋업 및 실행

> 표기 규칙: **[필수]** = 꼭 해야 함 / **[선택]** = 상황에 따라. 각 단계에 *용도*를 적음.
> ※ 백엔드 코드는 빌드 단계에서 생성됨. 이 매뉴얼대로 동작하게 맞춰 만듦.

**전제**: `00_사전_준비물_설치.md` 완료 (uv 설치 + Docker Desktop 실행 중).

---

## 처음 한 번 (순서대로)

### 1) 백엔드 폴더로 이동 — [필수]
```powershell
cd D:\work\project\workspace\Prj\stockbot\backend
```
*용도*: 이후 명령을 이 폴더에서 실행.

### 2) `.env` 만들기 — [필수]
```powershell
Copy-Item .env.example .env
# .env 열어서 값 입력 (PAIRING_KEY, TOSS_KEY_ENC_KEY 등 — 02 참고)
```
*용도*: 비밀·설정 로드. **없으면 서버 시작 실패.**

### 3) Postgres 띄우기 (도커) — [필수]
```powershell
docker compose up -d
```
*용도*: DB 기동. 없으면 마이그레이션·조회가 실패. (확인: `docker ps`)

### 4) 의존성 설치 + 가상환경 준비 — [필수]
```powershell
uv sync
```
*용도*: `.venv` 자동 생성 + 라이브러리 설치. (수동 venv 불필요)

### 5) DB 마이그레이션 적용 — [필수]
```powershell
uv run alembic upgrade head
```
*용도*: `credentials` 등 테이블 생성. **안 하면 DB에 테이블이 없어 연결·조회 실패.**

### 6) 서버 실행 — [필수]
```powershell
uv run uvicorn stockbot.main:app --reload --port 8000
```
*용도*: API 서버 기동. (`--reload` = 코드 바뀌면 자동 재시작, 개발용)

### 7) 동작 확인 — [선택(권장)]
- 브라우저: http://localhost:8000/health → 정상 응답
- API 문서: http://localhost:8000/docs

*용도*: 잘 떴는지 확인.

---

## 종료
```powershell
# 서버: 터미널에서 Ctrl + C
docker compose down      # [선택] Postgres 내리기 (데이터는 볼륨 유지)
```

## 이후 다시 실행할 때 (셋업 끝난 뒤)
```powershell
docker compose up -d                                   # [필수] (안 떠있으면)
uv run uvicorn stockbot.main:app --reload --port 8000  # [필수] 서버
```
- `uv sync`(4) / `uv run alembic upgrade head`(5)는 **[선택]** — 라이브러리·스키마가 바뀌었을 때만.

---

## 내가(사용자) 하는 것 vs 자동
- **내가**: 위 명령 실행 + `.env` 값 입력 + Docker Desktop 켜기.
- **자동(uv가)**: 가상환경 생성·라이브러리 설치·파이썬 버전 맞추기.
