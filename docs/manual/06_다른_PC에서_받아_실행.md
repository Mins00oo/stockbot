# 06. 다른 PC에서 받아 실행하기 (git clone 후)

> 핵심: **git에서 오는 것 / 안 오는 것**을 구분하면 끝. 표기: **[필수]**/**[선택]** + *용도*.

## git에 들어있는 것 (clone하면 그대로 옴)
- 소스코드 전부, `pyproject.toml`, **`uv.lock`**(백엔드 버전 잠금), `package.json`, **`package-lock.json`**(프론트 버전 잠금), `alembic/` 마이그레이션, `docker-compose.yml`, 매뉴얼.
- → 그래서 **버전은 어느 PC든 동일하게 재현**됨.

## git에 없는 것 (새 PC에서 직접 챙겨야) ⚠️
| 항목 | 왜 없나 | 어떻게 |
|---|---|---|
| **`.env`** (백·프) | 비밀이라 gitignore | 새로 만듦 (기존 값 **복사** 권장 / 또는 생성 — `02` 참고) |
| `.venv` / `node_modules` | 빌드 산출물 | `uv sync` / `npm install`이 자동 재생성 |
| **DB 데이터** | Postgres 볼륨은 git에 없음 | 새 PC는 **빈 DB** → 앱에서 토스 **재연결** |

---

## 백엔드 — 새 PC 절차
1. `git clone ...` 후 `cd stockbot\backend` — [필수]
2. `uv sync` — [필수] *용도*: `uv.lock`대로 동일 버전 설치(.venv 재생성)
3. `.env` 만들기 — [필수] *용도*: 비밀·설정. 기존 PC `.env` **값 복사** 권장.
   - ⚠️ **같은 DB를 쓸 거면 `TOSS_KEY_ENC_KEY`는 반드시 같은 값**(아니면 저장된 토스 키 복호화 불가).
4. `docker compose up -d` — [필수] *용도*: DB 기동 (새 PC면 빈 DB)
5. `uv run alembic upgrade head` — [필수] *용도*: 테이블 생성
6. `uv run uvicorn stockbot.main:app --reload --port 8000` — [필수] 서버
7. 앱에서 토스 **재연결** — [필수, *빈 DB일 때*] *용도*: 빈 DB엔 저장된 토스 키가 없음

## 프론트 — 새 PC 절차
1. `cd stockbot\frontend` — [필수]
2. `npm install` — [필수] *용도*: `package-lock.json`대로 설치
3. `npx expo install --fix` — [필수] *용도*: expo 패키지 SDK56 정렬
4. `.env` 만들기 (`EXPO_PUBLIC_API_URL`) — [필수]
5. dev build — [필수, 1회] (`05_프론트엔드_셋업_및_dev_build.md` 참고)

---

## 키 재사용 vs 재생성 (요약)
- **PAIRING_KEY**: 앱·백엔드 값만 일치하면 OK → **같은 값 복사** 권장.
- **TOSS_KEY_ENC_KEY**: 같은 DB 공유 → **동일 필수** / 빈 DB → 새로 OK(재연결).
- DB 데이터까지 옮기려면 → git 아님, `pg_dump` → 새 DB `restore` (별도 작업).
